# Changelog

`CHANGELOG.md` es la fuente principal de cambios del proyecto.

Este documento existe como referencia dentro de la documentacion modular y apunta al changelog raiz para evitar duplicacion o contradicciones.

## Fuente principal

- `../CHANGELOG.md`

## Regla

No mantener un changelog paralelo dentro de `docs/`.

Cuando se registre una version, mejora o Pull Request, actualizar `CHANGELOG.md`.
